# Contributing to Portable Pi 5 Cluster Server

Thank you for your interest in contributing! This project is designed for emergency communications and portable cluster computing, and we welcome improvements that align with these goals.

## Getting Started

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test thoroughly on your hardware setup
5. Submit a pull request with a clear description

## Guidelines

- **Documentation First:** Update [docs/](docs/) alongside any configuration changes
- **No Breaking Changes:** Preserve existing configuration file structures and command references
- **Hardware Agnostic:** Keep features compatible with Raspberry Pi 5 (4GB/8GB variants)
- **Testing:** Test changes on actual hardware when possible
- **Clear Commits:** Use descriptive commit messages

## Areas We're Looking For

- Hardware optimization and additional component support
- Security hardening improvements
- RF communication tool integration
- Mesh networking enhancements
- Performance improvements
- Documentation improvements

## Reporting Issues

Please open an issue with:
- Clear description of the problem
- Steps to reproduce (if applicable)
- Your hardware configuration
- Relevant error messages or logs

## Code Style

- Python scripts: Follow PEP 8 conventions
- Configuration files: Match existing format and structure
- Comments: Explain the "why", not just the "what"
